# Zebri Product Principles

These principles guide all product decisions.

------------------------------------------------------------------------

# 1. Simplicity First

If a feature makes the UI more complicated, it should not exist.

MCs should be able to learn the product in minutes.

------------------------------------------------------------------------

# 2. Faster Than Spreadsheets

Any workflow must be faster than using:

• Google Sheets • Notes • Email

If it isn't faster, redesign it.

------------------------------------------------------------------------

# 3. Focus on the Wedding Workflow

Everything should revolve around:

Lead → Booking → Wedding → Follow‑up

------------------------------------------------------------------------

# 4. Minimal Data Entry

Reduce typing wherever possible.

Prefer:

• dropdowns • defaults • quick actions

------------------------------------------------------------------------

# 5. One Screen = One Job

Each page should answer:

"What does the MC need to do right now?"

Avoid overloaded dashboards.

------------------------------------------------------------------------

# 6. Calm Interface

The UI should feel:

• quiet • focused • uncluttered

Whitespace is good.

------------------------------------------------------------------------

# 7. Opinionated Design

Don't build infinite configuration.

Provide smart defaults.

------------------------------------------------------------------------

# 8. Mobile Friendly

MCs often check details during events.

Pages must work on mobile.

------------------------------------------------------------------------

# 9. Speed Matters

Interactions should feel instant.

Use optimistic updates when possible.

------------------------------------------------------------------------

# 10. Build the Command Centre

Zebri should feel like:

The **control panel for running weddings**.
